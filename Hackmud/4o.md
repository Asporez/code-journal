To automate the usage of the `ast.t1_cracker` script in *hackmud*, you can create a simple script that loops through a list of targets and calls the public script automatically. Here's an outline of how to approach it:

1. **Define a list of targets**: You'll want a list of system locations (`#s.name.loc`) that you want to pass into the `tar` parameter.
2. **Automate the calls**: The script should iterate over the target list and invoke the public script for each one.
3. **Handle delays**: Since *hackmud* has a rate-limiting system, you'll want to add delays between calls to avoid triggering cooldowns or getting locked out.

Here's a simplified version of how this script could look:

```javascript
function (context, args) {
    var targets = ["#s.target1.loc", "#s.target2.loc", "#s.target3.loc"];  // Add more locations
    var results = [];
    
    for (var i = 0; i < targets.length; i++) {
        // Run the public script ast.t1_cracker for each target
        var res = args.call({
            script: "ast.t1_cracker",
            args: {
                tar: targets[i]
            }
        });
        
        // Collect the result for each target
        results.push({target: targets[i], result: res});
        
        // Optional: Add a delay to avoid rate-limiting issues (adjust as needed)
        if (i < targets.length - 1) {
            sleep(2000);  // Sleep for 2 seconds between calls
        }
    }
    
    return results;  // Return results so you can see how each call performed
}
```

### Key elements:
- **`targets` array**: This holds the locations you want to target. You can expand this list with as many targets as you need.
- **`args.call`**: This is how you invoke the public script (`ast.t1_cracker`) for each target.
- **`sleep()`**: Helps to add a delay between calls to avoid potential rate-limiting (you can adjust the sleep duration based on your needs).
